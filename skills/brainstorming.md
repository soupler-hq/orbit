# SKILL: Brainstorming & Spec Extraction
> Never start building until you know what you're building and why

## ACTIVATION
Auto-loaded at the start of any new project, milestone, or major feature.

## CORE PRINCIPLES
1. **Ruthless Prioritization**: Ask only questions where the answer fundamentally changes the plan.
2. **Assumption Surfacing**: State what you believe to be true early, so it can be corrected.
3. **Chunked Approval**: Present design in small, reviewable sections to prevent rework.
4. **Socratic Discovery**: Use questions to surface hidden complexity and challenge scope creep.

## PATTERNS

### The Brainstorm Protocol
1. **Intent Extraction**: Max 5 targeted questions on users, goals, and constraints.
2. **Assumption Mapping**: Explicitly list baseline assumptions for user validation.
3. **Design Walkthrough**: Incremental presentation of overview, data model, API, and phases.
4. **Spec Consolidation**: Produce a machine-readable project specification.

## CHECKLISTS

### Spec Readiness
- [ ] Core vision and user "job-to-be-done" defined
- [ ] Measurable success criteria for v1 established
- [ ] Hard constraints (tech, timeline, compliance) documented
- [ ] v1 vs v2 scope explicitly separated
- [ ] All open questions assigned owners and deadlines

## ANTI-PATTERNS
- **Question Bombing**: Asking 10+ questions at once, overwhelming the user.
- **Assumed Intent**: Building without validating the "why" behind a requirement.
- **All-at-Once Design**: Presenting a massive spec instead of chunked, approved sections.
- **Subjective Goals**: Using vague success criteria like "modern UI" or "fast performance."
- **Ignoring Constraints**: Designing a solution that violates a known tech or budget limit.
